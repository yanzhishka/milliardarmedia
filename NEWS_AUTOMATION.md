# Автоновости для Telegram

Система создаёт один черновик только по запросу администратора через Telegram-бота
и отправляет его в личный чат. Финальное сообщение для ручной пересылки приходит
только после нажатия `✅ Готово к пересылке`.

Начните с `/start`: бот откроет редакторскую панель с постоянными кнопками
`Новость`, `Черновики`, `Подкасты`, `Управление` и `Помощь`. Для поиска новости
достаточно нажать `Новость` → `Найти свежую новость`; расписаний и автопостинга нет.

## Что делает поток

1. Администратор отправляет боту `/news` и нажимает `📰 Найти свежую новость`.
2. Функция читает RSS-ленты, отсеивает ранее использованные ссылки и запрашивает
   у Groq короткую позитивную заметку.
3. Темы политики, войн, террора, преступлений, катастроф, болезней и конфликтов
   не допускаются инструкцией модели. При сомнении она должна вернуть `publish: false`.
4. Бот присылает фото, текст, исходную ссылку и кнопки `Готово к пересылке`,
   `Править`, `Пропустить`. Правка делается ответом на карточку черновика.
5. После подтверждения бот присылает отдельный чистый пост без кнопок в личный
   Telegram-чат. Его нужно вручную переслать в канал; существующий Telegram webhook
   автоматически перенесёт пересланный пост на сайт.

## Настройки Cloudflare Pages

Добавьте в **Settings → Environment variables** (в production и preview при необходимости):

| Переменная | Значение |
| --- | --- |
| `GROQ_API_KEY` | API-ключ Groq. |
| `NEWS_RUN_SECRET` | Длинная случайная строка, которая защищает внутренний запуск генератора. |
| `NEWS_RUN_URL` | Необязательно. Полный URL `/api/news-run`, нужен только если Telegram webhook и сайт работают на разных доменах. По умолчанию определяется автоматически. |
| `NEWS_REVIEW_CHAT_IDS` | Telegram ID получателей черновиков через запятую. Если не задана, берётся существующая `TELEGRAM_ADMIN_IDS`. |
| `TELEGRAM_BOT_TOKEN` | Уже используется текущим ботом. |
| `TELEGRAM_PREMIUM_EMOJI` | `true`, только если аккаунт — владелец бота — имеет Telegram Premium. Включает custom emoji в личных сообщениях бота. |
| `TELEGRAM_CHANNEL_ID` или `TELEGRAM_CHANNEL_USERNAME` | Уже используется текущим webhook для чтения постов канала. Бот должен остаться администратором для получения `channel_post`, но право на публикацию ему не нужно. |
| `PEXELS_API_KEY` | Необязательно. Бесплатный ключ Pexels используется только как резерв, если у первоисточника нет релевантного изображения. Бот отбрасывает баннеры, логотипы и служебные карточки, а выбранное фото загружает в Telegram файлом. |
| `NEWS_FEEDS` | Необязательно. JSON-массив RSS URL вместо встроенного набора. |
| `GROQ_MODEL` | Необязательно, по умолчанию `llama-3.3-70b-versatile`. |

## Премиум-эмодзи в тексте, подписи и пересылка

Бот добавляет в текст от одного до трёх смысловых custom Premium emoji из набора
канала: для добрых новостей, открытий, космоса, транспорта и бизнеса. Их количество
и категории выбирает модель по содержанию; случайных или не связанных с новостью
значков она не добавляет. Одновременно модель выбирает от одной до трёх важных
фраз, которые бот выделяет **жирным** в Telegram. Подпись всегда заканчивается
ссылкой `📱 Миллиардар`.
Для фирменного анимированного эмодзи в подписи используется id
`5330237710655306682`. Бот отправляет custom emoji только в личный чат редактора.
Если владелец бота имеет Telegram Premium, включите `TELEGRAM_PREMIUM_EMOJI=true`.

Пересылайте именно финальное сообщение, которое бот присылает после кнопки
`Готово к пересылке`. При пересылке Premium-аккаунтом custom emoji сохраняются.
Telegram может показать пометку о пересылке от бота; это нельзя убрать кодом бота.
Если клиент Telegram предлагает опцию «Скрыть имя отправителя» при пересылке в
канал, включите её и проверьте один тестовый пост. Пока Premium не настроен, бот
безопасно использует обычные emoji вместо custom emoji.

## Проверка

Напишите боту `/news`, нажмите `📰 Найти свежую новость` и дождитесь одного
черновика в личном Telegram-чате. Черновик содержит 2–3 абзаца объёмом от 280
до 720 знаков; если первый ответ модели короче, бот запрашивает развёрнутую
версию один раз.
