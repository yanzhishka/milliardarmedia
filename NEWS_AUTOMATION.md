# Автоновости для Telegram

Система создаёт один черновик только по запросу администратора через Telegram-бота
и отправляет его в личный чат. Финальное сообщение для ручной пересылки приходит
только после нажатия `✅ Готово к пересылке`.

Начните с `/start`: в редакторской панели есть одна прямая кнопка
`✍️ Написать пост`, а также отдельные разделы подкастов, управления и помощи.
Нажатие сразу запускает поиск и генерацию — подменю и списка черновиков нет.
Расписаний и автопостинга нет.

## Что делает поток

1. Администратор нажимает `✍️ Написать пост` или отправляет `/news`.
2. Функция читает RSS-ленты, отсеивает ранее использованные ссылки и запрашивает
   у Groq короткую новостную заметку.
3. Новость может быть позитивной или нейтральной. Политика запрещена полностью;
   также не допускаются войны, террор, преступления, насилие и тяжёлые трагедии.
   При прямом нарушении правил модель должна вернуть `publish: false`.
4. Бот присылает фото, текст, исходную ссылку и кнопки `Готово к пересылке`,
   `Править`, `Пропустить`. Правка делается ответом на карточку черновика.
5. После подтверждения бот присылает отдельный чистый пост без кнопок в личный
   Telegram-чат. Его нужно вручную переслать в канал; существующий Telegram webhook
   автоматически перенесёт пересланный пост на сайт.

## Настройки Cloudflare Pages

Добавьте в **Settings → Environment variables** (в production и preview при необходимости):

| Переменная | Значение |
| --- | --- |
| `GROQ_API_KEY` | API-ключ Groq. |
| `NEWS_RUN_SECRET` | Длинная случайная строка, которая защищает внутренний запуск генератора. |
| `NEWS_RUN_URL` | Необязательно. Полный URL `/api/news-run`, нужен только если Telegram webhook и сайт работают на разных доменах. По умолчанию определяется автоматически. |
| `NEWS_REVIEW_CHAT_IDS` | Telegram ID получателей черновиков через запятую. Если не задана, берётся существующая `TELEGRAM_ADMIN_IDS`. |
| `TELEGRAM_BOT_TOKEN` | Уже используется текущим ботом. |
| `TELEGRAM_PREMIUM_EMOJI` | `true`, только если аккаунт — владелец бота — имеет Telegram Premium. Включает фирменный custom emoji `📱` в шильдике канала; на эмодзи в тексте не влияет. |
| `TELEGRAM_CHANNEL_ID` или `TELEGRAM_CHANNEL_USERNAME` | Уже используется текущим webhook для чтения постов канала. Бот должен остаться администратором для получения `channel_post`, но право на публикацию ему не нужно. |
| `PEXELS_API_KEY` | Необязательно. Бесплатный ключ Pexels используется только как резерв, если у первоисточника нет релевантного изображения. Бот отбрасывает баннеры, логотипы и служебные карточки, а выбранное фото загружает в Telegram файлом. |
| `NEWS_FEEDS` | Необязательно. JSON-массив дополнительных RSS URL; они получают приоритет и добавляются к встроенному набору. Всего используется до 24 лент. |
| `GROQ_MODEL` | Необязательно, по умолчанию `llama-3.3-70b-versatile`. |

Встроенный набор начинается с российских и русскоязычных источников: Naked
Science, N+1, «ТехИнсайдер», iXBT, Хабр, «Афиша Daily», «Нож», DTF и GoHa.
Русскоязычные запросы Google News также остаются в ротации. Международные ленты
используются после них для расширения выбора.

## Диагностика

Раздел `Управление` → `Статус` выполняет живую проверку Telegram Bot API,
webhook, доступа бота к каналу и выбранной модели Groq. Там же показываются
очередь входящих обновлений, состояние KV, настройки Premium-шильдика и Pexels,
количество RSS-источников, постов и подкастов, дата последней публикации и
состояние её изображения. Кнопка `🔄 Обновить` повторяет все проверки.

## Эмодзи в тексте, Premium-шильдик и пересылка

Бот выбирает ровно один обычный эмодзи по контексту новости и ставит его только
после последнего предложения. В заголовке и остальных частях текста эмодзи не
используются. Даже если модель случайно вернёт дополнительные эмодзи, бот удалит
их перед отправкой. Одновременно модель выбирает от одной до трёх важных фраз,
которые бот выделяет **жирным** в Telegram.

Подпись всегда заканчивается ссылкой `📱 Миллиардар`. Только фирменный шильдик
остаётся Premium/custom emoji; для него используется id `5330237710655306682`.
Если владелец бота имеет Telegram Premium, включите `TELEGRAM_PREMIUM_EMOJI=true`.

Пересылайте именно финальное сообщение, которое бот присылает после кнопки
`Готово к пересылке`. При пересылке Premium-аккаунтом custom emoji шильдика сохраняется.
Telegram может показать пометку о пересылке от бота; это нельзя убрать кодом бота.
Если клиент Telegram предлагает опцию «Скрыть имя отправителя» при пересылке в
канал, включите её и проверьте один тестовый пост. Пока Premium не настроен, в
шильдике безопасно используется обычный `📱`; эмодзи в самом тексте всегда обычный.

## Проверка

Нажмите `✍️ Написать пост` и дождитесь результата в личном Telegram-чате. Пост
содержит от одного до трёх смысловых абзацев объёмом от 280 до 720 знаков:
простой инфоповод остаётся одним блоком, отдельный контекст и детали выносятся в
новые абзацы. Если первый ответ модели короче, бот запрашивает развёрнутую версию
один раз.
